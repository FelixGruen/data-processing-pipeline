class dpp.Pipeline(storage_name='', initialization_generator=None)[source]¶

A pipeline is not strictly needed, but offers a number of useful services. It should always be used in a with-block.

close()[source]¶

Close the pipeline and all objects that were registered to be closed.

has_value(my_id)[source]¶

Check if a value was stored for the given id.

initialize(init_func)[source]¶

Register an initialization function. The function will be supplied with datapoints from the initialization iterable of the pipeline once the pipeline definition is completed and the with-block is exited.

load_value(my_id)[source]¶

Return a stored value for the given id or None if no value was stored.

offers_initialization()[source]¶

Check whether the pipeline was supplied with an iterable that supplies datapoints for initialization.

register(name)[source]¶

Register a function with the pipeline. Returns an id that can be used to store values.

register_close(obj)[source]¶

Register an object that has to be closed after use with the pipeline, so that close is called on this object, when the pipeline is closed.

sign_up(generator)[source]¶

Sign up a generator as the current generator. This is used, so the pipeline can itself be used as an iterable.

store_value(my_id, value)[source]¶

Allows registered functions to store a value or object. This can be used to carry over values that were set during initialization into otherwise side-effect free transformation functions.

dpp.apply(source, transform)[source]¶

Turns a normal function into a generator and registers it with the current pipeline.

The function must accept whatever the given source yields and return a transformed datapoint based on its input.

Parameters:
  • source (iterable) – An iterable over a number of datapoints that transform accepts.
  • transform (function) – A function that transforms each datapoint that source yields.
Returns:

gen – A generator that yields the output of transform.
Return type:

generator
